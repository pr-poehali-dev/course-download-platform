# Инструкция по тестированию платежной системы Тинькофф

## Созданы файлы для тестирования:

### 1. simple-test.html
**Самый простой способ тестирования**

Откройте этот файл в браузере (двойной клик) и нажмите кнопку "Создать тестовый платеж".

Вы увидите:
- Запрос который отправляется
- Ответ от сервера
- Если успешно - ссылку на страницу оплаты
- Если ошибка - детали ошибки

### 2. test-payment.html
**Продвинутое тестирование с статистикой**

Открыть в браузере и:
- Нажать "Запустить 1 тест" для одиночного теста
- Нажать "Запустить 10 тестов" для серии из 10 попыток

Показывает статистику: сколько успешных, сколько ошибок.

### 3. test-token-generation.py
**Проверка генерации токена**

Запустить в терминале:
```bash
python3 test-token-generation.py
```

Показывает:
- Как генерируется токен шаг за шагом
- Итоговый токен
- Альтернативные варианты с демо-паролем

### 4. test-payment.sh
**Автоматическое тестирование через curl**

Запустить в терминале:
```bash
chmod +x test-payment.sh
./test-payment.sh
```

Выполнит 10 тестов подряд и покажет результаты.

## Что проверить:

1. **Откройте simple-test.html в браузере**
2. **Нажмите кнопку "Создать тестовый платеж"**
3. **Посмотрите на результат:**

   Если видите ERROR 204 "Неверный токен":
   - Возможно в Тинькофф админке есть отдельное поле "Секретный ключ" (не "Пароль")
   - Для DEMO терминалов обычно используется стандартный пароль: `wvchd3k3ju9vms5e`

4. **Проверьте логи backend:**
   После теста я могу посмотреть логи функции и увидеть детали генерации токена.
